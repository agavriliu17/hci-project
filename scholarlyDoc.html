1. Introduction

1.1 Purpose

This SRS specifies the requirements for the begr (best goods recommender) app, v1.0. The product's scope consists of an interactive application to be deployed on a smart device, able to recommend the best goods to buy based on quality, price and various restrictions. The app will be able to recommend goods based on the user's social graph.

1.3 Intended Audience and Reading Suggestions

This document is intended for the development team and for the teachers that are going to be grading the product and documentation.

1.4 Product Scope

The begr (best goods recommender) app is intended to help users find the best goods to buy based on quality, price and various restrictions. The app will be able to recommend goods based on the user's social graph.

2. Overall Description

2.1 Product Perspective

This product originates from one of the project proposals for the Human-Computer Interaction course, class 3A6. The app is intended to be used on a smart device, such as a smartphone or a tablet/fridge with a touchscreen.

A story board that describes the user's interaction with the app is presented below.

The sitemap of the app is presented below.

Some of the app wireframes are presented below.

2.2 Product Functions

  • Account features
    • Users can manage an account inside the app.
  • Product review features
    • Users can review products they have bought.
    • Users can see reviews of products they are interested in.
  • History features
    • Users can see their purchase history.
    • Users can see their review history.
  • Recommendation features
    • Users can see recommendations based on their purchase history.
    • Users can see recommendations based on their review history.
    • Users can see recommendations based on their social graph.
  • Search features
    • Users can search for products.
    • Users can filter products based on various criteria.
  • Product features
    • Users can see details about a product they are interested in by scanning its QR code.
    • Users can see the price of a product.
    • Users can see the quality of a product.
    • Users can see the quantity of a product.
    • Users can see the location of a product.
    • Users can see the reviews of a product.

2.3 Uses Classes and Characteristics

The app is intended to be used by people who want to buy goods, but don't know which ones are the best for them. The app will help them by recommending products based on their purchase history, review history and social graph. The app is intended to be used by people of all ages and it should be easy to use, with a simple and intuitive interface. The app should be accessible to people with no (or limited) computer knowledge.

2.4 Operating Environment

The app should be used on a smart device, such as a smartphone or a tablet/fridge with a touchscreen.

2.5 Design and Implementation Constraints

The app should be able to run on any smart device, such as a smartphone or a tablet/fridge with a touchscreen. The app should be easy to use, with a simple and intuitive interface.

2.6 User Documentation

A relevant guide to using the app will be provided in the form of user flows available in the Figma prototype and in the form of a video demo.

2.7 Assumptions and Dependencies

The app is not intended to have an implementation. It is just a project proposal for the Human-Computer Interaction course. The design of the app is described in the following sections.

3. External Interface Requirements

3.1 User Interface

A user can interact in the Graphical User Interface with the following pages:

  • Home Page
  • Friends Page
  • Scan Page

3.1.1 Home Page

On the mobile version home page, there is a search bar that can be used to search for products. The search bar is followed by a list of products reviewed by the user's friends. The bottom of the page contains a navigation bar with the following buttons: Home, Friends, Scan, History and Settings.

On the tablet/fridge version home page, the design is similar to the mobile, but it contains a form for posting a review for the lastest product bought by the user and a history of the previously purchased products and their rating.

3.1.2 Social Page

On the mobile version social page, there is a horizontal list of friends. The list is followed by a form for adding a review for the lastest product bought by the user. The following is a list of reviews posted by the user's friends. The bottom of the page contains the same navigation bar.

On the tablet/fridge version social page, the design is similar to the mobile, but it is displayed differently.

3.1.3 Scan Page

On the mobile version scan page, the camera is used to scan the QR code of a product. The bottom of the page contains the same navigation bar.

After the QR code is scanned, the user is redirected to the product page. The product page contains the rating of the product, a tab with similiar products (the best reviwed product is at the top of the list) and a tab with reviews of the product. The bottom of the page contains the same navigation bar.

On the tablet/fridge version scan page, the design is similar to the mobile.

After the QR code is scanned, the user is redirected to the product page. The product page is similar to the mobile version.

3.1.4 History Page

On the mobile version history page, there are two tabs: one for browsing the history of the user's purchases and one for browsing the history of the user's reviews. The bottom of the page contains the same navigation bar.

On the tablet/fridge version history page, the design is similar to the mobile version.

3.1.5 Settings Page

On the mobile version settings page, the user can view the number of products they have scanned, the number of reviews they have posted, their name and email. They have the option to choose to post reviews anonymously. They can also choose to give permission to the app to learn from their reviews and recommend products to them. The bottom of the page contains the same navigation bar.

On the tablet/fridge version settings page, the design is similar to the mobile version.

4. System Features

4.1 Account Management

This high priority feature provides users the capability to login if they have an account or sign up to the web application by providing personal data. This is useful for the application to be able to provide a personalized experience for each user, such as displaying the products that the user has previously bought and rated, or displaying the reviews of the user's friends.

4.2 Product Management

This high priority feature provides users the capability to scan the QR code of a product in order to view its rating and reviews and receive recommendations for better rated products.

4.3 Social

This medium priority feature provides users the capability to view the reviews of their friends and add reviews for the products they have bought.

4.4 History

This low priority feature provides users the capability to view the products they have previously bought and rated.

4.5 Recommendation

This low priority feature provides users the capability to receive recommendations for better rated products.

5. Other Nonfunctional Requirements

5.2 Safety Requirements

The app should be safe to use. The app should not store any sensitive data about the user. The app should not contain any malicious code. The app should not contain any content that could be considered offensive.

5.3 Security Requirements

The app should be secure. The app should not store any sensitive data about the user.

There are no software quality attributes imposed. The app should follow the best practices for the technologies used.
